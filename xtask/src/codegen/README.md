# 关于 codegen 的一些细节说明

## 物品 ID

因为原版涉及到的物品已经几乎不会再变动，所以我们只需要在原有的原版物品 ID 的基础上做更新即可（几乎全都是直接添加，而不会覆盖）。

### 文件

以下是涉及到的文件：

- `item_ids.yml`: 原版（包含 DLC）的物品
- `item_ids_cer.yml`: 法魂（CER）的物品

这两个文件都遵循相同的数据格式，例如：

```yml
一级菜单:
  物品1: <十进制整数>
另一个一级菜单:
  二级菜单:
    物品2: <十进制整数>
# 通常只会有最多两级菜单，且二级菜单一般用于对武器的进一步分类
```

### 获取来源

soarqin 创建的 [ConvergenceER_CN](https://github.com/soarqin/ConvergenceER_CN) 仓库为 CER 中的物品、装备、护符等做了翻译。

### 解析方式

上述仓库的翻译中，对于这些东西的 ID 均使用了“偏移量+中文翻译”的形式，例如：

```text
= 2510000:"“战士壶”亚历山大"
= 2510001:"“战士壶”亚历山大＋１"
```

所有以 `=` 开头的行才是所需要的。其中前面的数字 ID 表示基于该事物类别（category）的基 ID 所产生的位移。以下是各种类别的基 ID：

- 武器：`0x0`
- 防具：`0x10000000`
- 护符：`0x20000000`
- 物品（道具、骨灰等）：`0x40000000`
- 战灰：`0x80000000`

对于武器，正常来说 +0 等级无质变时，武器的 ID 必然可以整除 `10000`（十进制），而翻译文本会包含有质变的武器，因此需要剔除这些数据。
